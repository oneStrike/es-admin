## 问题分析
当关闭 `es-table-select` 组件的弹窗时，父弹窗也会关闭。这是因为 Vben 的 `useVbenModal` 存在事件传播问题，子弹窗的 `onOpenChange` 事件会传递给父弹窗。

## 解决方案
按照用户要求，将组件中的 Vben 弹窗替换为 Element Plus 的 `ElDialog` 组件，避免事件传播问题。

## 修复步骤
1. **修改 es-table-select.vue 文件**
   - 移除 Vben 弹窗相关导入和使用
   - 导入 Element Plus 的 Dialog 组件
   - 添加弹窗状态管理
   - 使用 ElDialog 替代原有弹窗

2. **具体修改内容**
   - 移除 `useVbenModal` 导入
   - 导入 `ElDialog`, `ElDialogHeader`, `ElDialogTitle`, `ElDialogContent`, `ElDialogFooter`
   - 添加 `isDialogVisible` 状态变量
   - 实现 `openTableModal` 和 `closeTableModal` 方法
   - 使用 ElDialog 组件替代 Modal 组件
   - 调整事件处理：使用 ElDialog 的 `@opened` 事件替代原有 `onOpened` 回调

3. **修改原因**
   - 按照用户要求使用 Element Plus 弹窗
   - ElDialog 组件独立管理状态，不会出现事件传播问题
   - 每个弹窗的打开/关闭状态相互独立，不会影响父弹窗

## 修复效果
修复后，关闭 `es-table-select` 组件的弹窗时，只会关闭自身，不会影响父弹窗的状态，解决了父弹窗意外关闭的问题。同时，保持了组件的原有功能和使用方式不变。
