# ES Admin 规范总览

本文件是 `es-admin` 仓库内规范的索引入口。专项规范按功能拆分，`AGENTS.md` 只保留项目级最小约束、决策顺序与验证基线。

## 决策顺序

1. 用户或需求方在当前任务中的明确指令。
2. 根 `AGENTS.md` 的项目级最小约束与验证基线。
3. 本目录下各专项规范文件。
4. 仓库现有自动化配置、脚本与同域稳定实现。
5. 根目录 README 与各子目录 README。

若规范与当前可运行代码、生成接口契约、Vben 基座现实或部署约束冲突，以兼容性优先，并在交付说明中记录冲突点与暂行决策。

## 维护原则

- 专项规范是事实源；不要在 `AGENTS.md` 重新维护同一批细则。
- 新增规范时优先补充现有专项文件；只有职责明显独立时才新增规则文件。
- 规范应服务当前 `apps/web-ele`、Vben、VxeGrid、生成接口与业务 model 的真实工作流，不照搬其他仓库的后端规则。
- 规范调整应保持可审查、可执行、可验证；禁止用大段口号替代具体约束。
