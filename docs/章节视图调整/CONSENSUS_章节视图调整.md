# 章节视图调整 - 共识文档

## 明确需求描述

将作品章节管理视图的实现方式与作品管理视图对齐，严格按照接口文档调整代码结构和字段配置，确保代码风格一致、功能完整。

## 技术实现方案

### 1. 表格列配置优化 (`columns.ts`)
- 参考作品管理的列配置模式
- 优化列的 `sort` 顺序属性
- 添加 `viewRule` 字典映射显示
- 调整列宽度和固定列设置
- 确保与 `ChapterPageResponse` 返回字段匹配

### 2. 表单配置优化 (`form.ts`)
- 检查字段与 `CreateWorkChapterDto` / `UpdateWorkChapterDto` 的一致性
- 确认 `viewRule` 默认值为 -1（继承）
- 确保 `workType` 在提交时正确设置（漫画=1）
- 验证依赖关系（dependencies）配置正确

### 3. 详情配置优化 (`detail.ts`)
- 保持分组卡片结构
- 确保字段与接口返回一致
- 优化 `viewRule` 的显示文本映射

### 4. 视图主文件优化 (`index.vue`)
- 代码风格与作品管理保持一致
- 类型导入使用明确的类型名
- 函数命名和注释风格统一

## 技术约束

- 使用项目现有的 Vben Admin 框架组件
- 保持 TypeScript 严格类型检查
- 复用现有的 hooks 和工具函数
- 不引入新的依赖

## 集成方案

章节管理作为模态框组件，通过 `workId` 和 `workName` 接收父组件传入的数据，与作品管理保持一致的通信方式。

## 任务边界限制

- 仅调整 `chapter/` 目录下的文件
- 不修改 API 层代码
- 不修改路由配置
- 不修改其他视图
