  - 签到管理：path `check-in`（相对路径，归属 `/user-manager`），name `CheckIn`

## 路由-视图一致性补齐策略（不改层级）

- 全量检查：遍历 `apps/web-ele/src/router/routes/modules/**/*.ts`，收集所有 `component: () => import('#/views/...')` 的视图路径
- 视图缺失判定：若目标路径不存在对应 `.vue` 文件，则纳入“视图补齐清单”
- 补齐原则（计划，不立即执行）：
  - 保持原有父级（不调整层级），在既定目录下创建最简占位视图文件
  - 占位视图统一结构：`<Page auto-content-height>` + 明确标题文案（可取自 `meta.title`）
  - 命名规范：文件名统一为 `index.vue`，目录用小写短横线命名（如 `check-in`、`level-rules`）

## 不合理路由配置调整原则（不改层级）

- 允许的调整（保持父级不变）：
  - 子路由 `path` 使用相对路径（不以 `/` 开头），确保 URL 归属父级层级，例如 `UserManager` 下使用 `path: 'profile'`
  - 去重 `name` 与 `path`，确保唯一性与可读性
  - `component` 指向真实存在的视图文件
  - `meta.title` 与业务语义一致（如“会员管理/签到管理/任务管理”）
- 不允许的调整：
  - 变更父级路由位置或引入新的父级层级
  - 将子路由提升为根级（避免出现 `/members` 等根路径）
