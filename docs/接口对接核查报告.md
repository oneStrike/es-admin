## 2. 本次已直接修复的问题

### 2.1 全局分页页码映射错误

- 问题：OpenAPI 类型中大量分页请求字段明确为 `pageIndex` 从 `0` 开始，但项目内多个页面直接将表格页码 `currentPage` 原样传给后端
- 修复：
  - 在 `src/adapter/vxe-table.ts` 新增统一转换方法
  - `formatQuery` 已改为统一提交 `0` 基 `pageIndex`
  - 手写分页页面也已补齐转换
- 影响范围：
  - 统一适配的页面：`agreement`、`user-manager/profile`、`user-growth/points`、`user-growth/experience`、`user-growth/level-rules`、`user-growth/badges`、`system-manager/tasks`、`system-manager/user-manager`、`system-manager/data-dictionary/item`、`forum/topic`、`forum/tags`、`forum/sections`、`forum/moderators`、`forum/moderator-application`、`content-manager/comic-manager/core`、`content-manager/comic-manager/chapter`、`content-manager/novel-manager/core`、`content-manager/novel-manager/chapter`、`content-manager/tag-manager`、`content-manager/category-manager`、`content-manager/author-manager`、`task-assignment-modal`、`user-operation-modal`
  - 手写修复页面：`app-manager/page-manager`、`app-manager/announcement`、`system-manager/profile`、`forum/sensitive-word`、`system-manager/data-dictionary`、`system-manager/log-manager/login-log`、`system-manager/log-manager/operation-log`
  - 特殊修复页面：`content-manager/comic-manager/third-party`，同时补了返回页码 `0` 基转 `1` 基

### 2.2 公告查询时间字段映射错误

- 页面：`app-manager/announcement`
- 问题：搜索表单使用字段 `dateTimeRange`，但查询时原样透传给分页接口，后端真实字段为 `publishStartTime` / `publishEndTime`
- 修复：
  - 查询前已拆分 `dateTimeRange`
  - 已改为向接口提交 `publishStartTime`、`publishEndTime`
  - 同时保留 `enablePlatform` 的 JSON 字符串转换

### 2.3 公告结束时间显示与过期判定不一致

- 页面：`app-manager/announcement`
- 问题：
  - 详情页展示把结束时间固定拼成 `23:59:59`
  - 过期判断直接使用原始 `publishEndTime`
  - 如果后端返回的是日期字符串，展示与逻辑会出现同一天内不一致
- 修复：
  - 新增统一结束时间归一化处理
  - 详情展示改为统一格式化结果
  - 发布状态判断改为按归一化后的结束时间判断

### 2.4 页面管理权限枚举错误

- 页面：`app-manager/page-manager`
- 问题：前端将 `accessLevel = 3` 显示为“管理员访问”
- 类型基准：`src/api/types/appPage.d.ts` 定义为 `3 = VIP`
- 修复：已改为“VIP访问”

### 2.5 论坛板块审核策略枚举错误

- 页面：`forum/sections`
- 问题：前端只有 `0/1/2` 三个选项，且语义为“无需审核 / 人工审核 / 自动审核”
- 类型基准：`src/api/types/forum.d.ts` 定义为：
  - `0 = NONE`
  - `1 = SEVERE_SENSITIVE_WORD`
  - `2 = GENERAL_SENSITIVE_WORD`
  - `3 = MILD_SENSITIVE_WORD`
  - `4 = MANUAL`
- 修复：已扩展为完整五档策略，并同步用于表单、列表展示、详情展示

### 2.6 敏感词等级枚举错误

- 页面：`forum/sensitive-word`
- 问题：前端原先配置为 `1=低, 2=中, 3=高`
- 类型基准：`src/api/types/forum.d.ts` 定义为 `1=SEVERE, 2=GENERAL, 3=LIGHT`
- 修复：已调整为 `1=高, 2=中, 3=低`，颜色也同步修正
